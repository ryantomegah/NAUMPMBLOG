Optimising images is crucial for web performance, and Next.js makes it easy with its built-in `next/image` component. In this article, you'll learn how to use it and best practices for image optimisation in your Next.js apps.

![Laurier logo](https://ryan-naum.vercel.app/laurier.png)

## Why Image Optimisation Matters

Large, unoptimised images can slow down your website, increase bounce rates, and hurt SEO. By optimising images, you ensure faster load times and a better user experience.

## Using the `next/image` Component

Next.js provides the `Image` component, which automatically optimises images on-demand:

```jsx
import Image from "next/image";

export default function Example() {
  return <Image src="/public/atomic.png" alt="Atomic logo" width={400} height={300} priority />;
}
```

- **Automatic resizing**: Serve images at the right size for every device.
- **Lazy loading**: Images load only when they enter the viewport.
- **Modern formats**: Next.js can serve WebP or AVIF when supported.

## Best Practices

| Practice               | Description                                     |
| ---------------------- | ----------------------------------------------- |
| Use `next/image`       | Always use the built-in component for images.   |
| Specify width & height | Prevents layout shift and improves performance. |
| Optimise source files  | Use tools like TinyPNG before uploading.        |
| Use CDN                | Leverage Vercel or a custom CDN for delivery.   |

## Example Output

```jsx
<Image src="/public/atomic.png" alt="Atomic logo" width={400} height={300} />
```

By following these tips and using Next.js features, your site will be faster and more user-friendly.
